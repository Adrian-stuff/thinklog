# Product

## Register

product

## Users

Software developers and engineers who want to stay informed about engineering best practices, AI/ML research, cloud computing, and web development. They are tired of noise and want curated, high-quality technical content in a fast, focused reading experience.

## Product Purpose

ThinkLog aggregates and curates developer-focused articles from top engineering blogs into a single performant feed with social features (reactions, comments, search, sharing). It replaces scattered newsletter subscriptions and social media feeds with one intentional reading space.

## Brand Personality

Professional, minimal, sharp. Terminal-adjacent but not literal. The feel of a well-designed CLI tool: fast, responsive, nothing wasted. Confident without being arrogant.

## Anti-references

Do not look like a generic SaaS dashboard. No heavy shadows, overdesigned cards, or generic enterprise UI patterns. No Medium clone (no clap buttons, no centered narrow columns). Avoid visual clutter that distracts from reading.

## Design Principles

1. **Content first, chrome last.** Every pixel that isn't content should earn its place. Navigation, controls, and UI chrome are secondary to the reading experience.
2. **Speed is a feature.** The interface should feel as fast as the RSS fetching is automated. No wasteful animations that delay task completion.
3. **Developer-native, not generic.** The aesthetic should resonate with developers without being gimmicky. Think professional tools (Linear, Raycast, Warp) not themed novelty.
4. **Subtle feedback, clear state.** Interactions acknowledge themselves without demanding attention. A button press, a toggle, a load state — all communicate without fanfare.

## Accessibility & Inclusion

No special requirements beyond basic good practices. Reduced motion support should be included for users who prefer it.
